# P2P 局域网同步方案 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在同一 WiFi/LAN 下，两台设备无需任何服务器即可直接同步加密配置（支持桌面 ↔ 桌面、桌面 ↔ Android）。

**Architecture:** 一台设备启动临时 HTTP 服务（"分享模式"），另一台设备通过局域网 IP + 端口连接，拉取或推送加密配置。通过二维码传递连接地址，复用现有的 RSA-2048-OAEP + AES-256-GCM 混合加密。服务只在同步期间运行，同步完成后自动关闭。

**Tech Stack:** Rust + Tauri 2.0 + tiny-http（轻量 HTTP 服务）+ reqwest（HTTP 客户端）+ qrcode（二维码生成）+ 本地网络 IP 探测

---

## 背景与方案对比

### 为什么需要 P2P 局域网同步

现有同步方案：
- **URL 拉取**：单向、手动操作多
- **Gist 同步**：需 GitHub 账号、需联网、推送有延迟

P2P 局域网同步的价值：
- ✅ 完全离线可用（无互联网也能同步）
- ✅ 零依赖（不需要任何账号或服务）
- ✅ 速度快（局域网千兆带宽）
- ✅ 隐私性强（数据不出局域网）

### 适用场景

```
桌面端 ↔ 手机     同一 WiFi 下快速同步
桌面端 ↔ 桌面端   两台电脑之间快速传配置
没有网络时         纯离线环境下也能同步
```

## 设计方案

### 整体流程

```
┌─────────────┐          LAN          ┌─────────────┐
│  设备 A      │  HTTP GET / POST      │  设备 B      │
│  (客户端)    │ ──────────────────►  │  (服务端)    │
│             │ ◄──────────────────  │             │
└─────────────┘   加密配置 JSON       └─────────────┘
       ▲                                    ▲
       │                                    │
       └────────── 本地加密/解密 ───────────┘
                  （复用现有 sync 模块）
```

### 两种模式

**1. 分享模式（服务端）**
- 启动 HTTP 服务，监听 `0.0.0.0:{随机端口}`
- 暴露 `GET /sync` → 返回加密配置 JSON
- 暴露 `POST /sync` → 接收加密配置 JSON 并写入本地
- 显示本机 IP + 端口 + 二维码
- 超时或同步完成后自动关闭服务

**2. 连接模式（客户端）**
- 输入对端 IP:port（或扫码）
- 拉取：GET 对端 `/sync` → 解密 → 合并/覆盖
- 推送：加密本地配置 → POST 对端 `/sync`

### 安全设计

- **传输层**：HTTP 明文传输的是**加密后的** `EncryptedPayload`（RSA-2048-OAEP + AES-256-GCM）
- **应用层**：接收方需要私钥才能解密，中间人拿到密文也没用
- **服务发现**：不做自动广播，避免暴露服务存在，用户手动输入或扫码
- **自动关闭**：服务端 5 分钟无操作自动关闭，或同步完成后立即关闭
- **绑定地址**：监听 `0.0.0.0` 但只在局域网可达，不暴露到公网

```rust
//! P2P 局域网同步模块
//!
//! 提供临时 HTTP 服务，用于局域网内设备之间直接同步加密配置。
//! 传输的是加密后的 payload（RSA-2048-OAEP + AES-256-GCM），
//! 即使在局域网内被嗅探也无法解密。

use std::net::TcpListener;

/// 获取本机局域网 IPv4 地址
///
/// 优先返回 WiFi/以太网的局域网地址（192.168.x.x / 10.x.x.x / 172.16-31.x.x）
pub fn get_local_ip() -> Result<String, String> {
    local_ip_address::local_ip()
        .map(|ip| ip.to_string())
        .map_err(|e| format!("获取本机 IP 失败: {}", e))
}

/// 找到一个可用的随机端口（30000-40000 之间）
pub fn find_available_port() -> Result<u16, String> {
    for port in 30000..40000 {
        if TcpListener::bind(("0.0.0.0", port)).is_ok() {
            return Ok(port);
        }
    }
    Err("找不到可用端口".into())
}

/// 生成二维码 PNG 数据（Base64 编码）
///
/// 输入要编码的字符串，返回 `data:image/png;base64,xxx` 格式的 data URL
pub fn generate_qrcode_data_url(content: &str) -> Result<String, String> {
    use qrcode::QrCode;
    use qrcode::render::unicode;

    let code = QrCode::new(content.as_bytes())
        .map_err(|e| format!("生成二维码失败: {}", e))?;

    // 渲染为 PNG
    let image = code.render::<image::Luma<u8>>()
        .min_dimensions(256, 256)
        .dark_color(image::Luma([0u8]))
        .light_color(image::Luma([255u8]))
        .build();

    // 编码为 PNG，再 Base64
    let mut png_buf = Vec::new();
    let mut cursor = std::io::Cursor::new(&mut png_buf);
    image.write_to(&mut cursor, image::ImageFormat::Png)
        .map_err(|e| format!("PNG 编码失败: {}", e))?;

    use base64::{Engine as _, engine::general_purpose::STANDARD as B64};
    let b64 = B64.encode(&png_buf);
    Ok(format!("data:image/png;base64,{}", b64))
}
```

等等——`qrcode` crate 的 PNG 渲染需要 `image` crate。我们再检查一下。

**修正 Step 1**：`qrcode` 的 PNG 渲染需要 `image` 依赖。让我改一下，用更轻量的方式。

用 `qrcode` + 手动生成 SVG（不需要 `image` 依赖，体积更小）：

```rust
/// 生成二维码 SVG 字符串
///
/// 返回完整的 SVG 字符串，前端直接渲染
pub fn generate_qrcode_svg(content: &str) -> Result<String, String> {
    use qrcode::QrCode;

    let code = QrCode::new(content.as_bytes())
        .map_err(|e| format!("生成二维码失败: {}", e))?;

    // 手动构建 SVG
    let size = code.width();
    let scale = 8;
    let total_size = size * scale;

    let mut svg = String::new();
    svg.push_str(&format!(
        r#"<svg xmlns="http://www.w3.org/2000/svg" width="{}" height="{}" viewBox="0 0 {} {}">"#,
        total_size, total_size, total_size, total_size
    ));
    // 白色背景
    svg.push_str(&format!(
        r#"<rect width="{}" height="{}" fill="white"/>"#,
        total_size, total_size
    ));

    for y in 0..size {
        for x in 0..size {
            if code[(x, y)] == qrcode::Color::Dark {
                svg.push_str(&format!(
                    r#"<rect x="{}" y="{}" width="{}" height="{}" fill="black"/>"#,
                    x * scale,
                    y * scale,
                    scale,
                    scale
                ));
            }
        }
    }

    svg.push_str("</svg>");
    Ok(svg)
}
```

好，用 SVG 方案，不需要额外的 `image` 依赖，体积小得多。

- [ ] **Step 1: 在 `p2p.rs` 中添加 HTTP 服务端**

在文件底部追加：

```rust
use std::sync::{Arc, Mutex};
use std::thread;
use std::time::Duration;
use tiny_http::{Server, Response, Header, Method, StatusCode};

/// P2P 同步服务端状态
pub struct P2PServer {
    /// 监听地址（ip:port）
    pub addr: String,
    /// 服务是否正在运行
    running: Arc<Mutex<bool>>,
    /// 加密后的配置 payload（服务端分享的内容）
    payload_json: Arc<Mutex<Option<String>>>,
}

impl P2PServer {
    /// 启动 P2P 同步服务
    ///
    /// - `ip`: 本机局域网 IP
    /// - `port`: 监听端口
    /// - `initial_payload`: 初始加密配置（可以为空，后续通过 set_payload 更新）
    /// - `on_receive`: 当收到客户端 POST 过来的新 payload 时的回调
    pub fn start<F>(
        ip: &str,
        port: u16,
        initial_payload: Option<String>,
        mut on_receive: F,
    ) -> Result<Self, String>
    where
        F: FnMut(String) + Send + 'static,
    {
        let addr = format!("{}:{}", ip, port);
        let server = Server::http(&addr)
            .map_err(|e| format!("启动 HTTP 服务失败 ({}): {}", addr, e))?;

        let running = Arc::new(Mutex::new(true));
        let payload_json = Arc::new(Mutex::new(initial_payload));

        let running_clone = running.clone();
        let payload_clone = payload_json.clone();

        // 后台线程运行服务
        thread::spawn(move || {
            for request in server.incoming_requests() {
                // 检查是否应该停止
                if !*running_clone.lock().unwrap() {
                    let _ = request.respond(
                        Response::from_string("server stopped").with_status_code(StatusCode(503))
                    );
                    break;
                }

                let method = request.method().clone();
                let url = request.url().to_string();

                match (method, url.as_str()) {
                    (Method::Get, "/sync") => {
                        // GET /sync → 返回加密配置
                        let payload = payload_clone.lock().unwrap().clone();
                        match payload {
                            Some(json) => {
                                let resp = Response::from_string(json)
                                    .with_header(
                                        Header::from_bytes(
                                            b"Content-Type",
                                            b"application/json; charset=utf-8"
                                        ).unwrap()
                                    )
                                    .with_header(
                                        Header::from_bytes(
                                            b"Access-Control-Allow-Origin",
                                            b"*"
                                        ).unwrap()
                                    );
                                let _ = request.respond(resp);
                            }
                            None => {
                                let resp = Response::from_string(
                                    r#"{"error":"no payload shared yet"}"#
                                ).with_status_code(StatusCode(404))
                                 .with_header(
                                    Header::from_bytes(
                                        b"Content-Type",
                                        b"application/json"
                                    ).unwrap()
                                 )
                                 .with_header(
                                    Header::from_bytes(
                                        b"Access-Control-Allow-Origin",
                                        b"*"
                                    ).unwrap()
                                 );
                                let _ = request.respond(resp);
                            }
                        }
                    }

                    (Method::Post, "/sync") => {
                        // POST /sync → 接收加密配置并触发回调
                        let mut body = String::new();
                        let _ = request.as_reader().read_to_string(&mut body);

                        if !body.is_empty() {
                            on_receive(body.clone());
                            // 同时更新本地分享的 payload
                            *payload_clone.lock().unwrap() = Some(body);
                        }

                        let resp = Response::from_string(r#"{"ok":true}"#)
                            .with_header(
                                Header::from_bytes(
                                    b"Content-Type",
                                    b"application/json"
                                ).unwrap()
                            )
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Origin",
                                    b"*"
                                ).unwrap()
                            );
                        let _ = request.respond(resp);
                    }

                    (Method::Options, _) => {
                        // CORS 预检
                        let resp = Response::empty(StatusCode(204))
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Origin",
                                    b"*"
                                ).unwrap()
                            )
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Methods",
                                    b"GET, POST, OPTIONS"
                                ).unwrap()
                            )
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Headers",
                                    b"Content-Type"
                                ).unwrap()
                            );
                        let _ = request.respond(resp);
                    }

                    (Method::Post, "/stop") => {
                        // POST /stop → 停止服务（用于前端主动停止）
                        let resp = Response::from_string(r#"{"ok":true}"#)
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Origin",
                                    b"*"
                                ).unwrap()
                            );
                        let _ = request.respond(resp);
                        *running_clone.lock().unwrap() = false;
                        break;
                    }

                    _ => {
                        let resp = Response::from_string(r#"{"error":"not found"}"#)
                            .with_status_code(StatusCode(404))
                            .with_header(
                                Header::from_bytes(
                                    b"Access-Control-Allow-Origin",
                                    b"*"
                                ).unwrap()
                            );
                        let _ = request.respond(resp);
                    }
                }
            }
        });

        Ok(Self {
            addr,
            running,
            payload_json,
        })
    }

    /// 更新分享的加密 payload
    pub fn set_payload(&self, payload: String) {
        *self.payload_json.lock().unwrap() = Some(payload);
    }

    /// 停止服务
    pub fn stop(&self) {
        *self.running.lock().unwrap() = false;
    }

    /// 检查服务是否还在运行
    pub fn is_running(&self) -> bool {
        *self.running.lock().unwrap()
    }
}
```

```rust
/// 启动 P2P 局域网同步服务
///
/// 返回 (addr, qrcode_svg)
/// - addr: 服务地址（ip:port）
/// - qrcode_svg: 包含地址的二维码 SVG 字符串
#[tauri::command]
fn p2p_start_server(app: tauri::AppHandle) -> Result<(String, String), String> {
    // 如果已有服务在跑，先停掉
    {
        let mut guard = p2p_server().lock().unwrap();
        if let Some(server) = guard.take() {
            server.stop();
        }
    }

    let ip = p2p::get_local_ip()?;
    let port = p2p::find_available_port()?;

    // 准备初始 payload（加密后的本地配置）
    let ss = state::load_sync_state(&app);
    let initial_payload = if let (Some(pub_key), _) = (ss.public_key_pem.as_ref(), ss.private_key_pem.as_ref()) {
        let cfg = config::load_config(&app);
        match sync::encrypt_config_to_string(pub_key, &cfg) {
            Ok(json) => Some(json),
            Err(_) => None,
        }
    } else {
        None
    };

    // 启动服务
    let app_handle = app.clone();
    let server = p2p::P2PServer::start(
        &ip,
        port,
        initial_payload,
        move |received_json| {
            // 收到远程推送的 payload → 存到一个临时文件，前端来取
            // 简单起见，我们写入 sync 状态的一个临时字段？
            // 或者直接发事件给前端
            let _ = app_handle.emit("p2p-data-received", received_json);
        },
    )?;

    let addr = server.addr.clone();

    // 生成二维码（内容是同步 URL）
    let url = format!("http://{}/sync", addr);
    let qrcode_svg = p2p::generate_qrcode_svg(&url)?;

    // 保存服务实例
    {
        let mut guard = p2p_server().lock().unwrap();
        *guard = Some(server);
    }

    Ok((addr, qrcode_svg))
}
```

```rust
/// 重新加密本地配置并更新分享内容（修改配置后调用）
#[tauri::command]
fn p2p_refresh_payload(app: tauri::AppHandle) -> Result<(), String> {
    let guard = p2p_server().lock().unwrap();
    let server = match guard.as_ref() {
        Some(s) => s,
        None => return Err("P2P 服务未启动".into()),
    };

    let ss = state::load_sync_state(&app);
    let pub_key = ss.public_key_pem.ok_or("未配置公钥")?;
    let cfg = config::load_config(&app);
    let payload = sync::encrypt_config_to_string(&pub_key, &cfg)?;
    server.set_payload(payload);

    Ok(())
}
```

```javascript
// ===== P2P 局域网同步 =====
async function p2pStartShare() {
  if (!syncState?.public_key_pem) {
    toast("请先配置公钥（用于加密分享内容）", "error");
    return;
  }

    // 监听对端推送过来的数据
    const unlisten = await appWindow.listen("p2p-data-received", (event) => {
      p2pHandleReceived(event.payload);
    });
    p2pUnlistenFn = unlisten;
  } catch (e) {
    toast("启动失败: " + (e?.message || e), "error");

async function p2pStop() {
  try {
    await invoke("p2p_stop_server");
    document.getElementById("p2pShareArea").style.display = "none";
    document.getElementById("p2pConnectArea").style.display = "";
    toast("已停止分享");
    syncLog("P2P 分享已停止", "info");
    if (p2pUnlistenFn) { p2pUnlistenFn(); p2pUnlistenFn = null; }
  } catch (e) {
    toast("停止失败: " + (e?.message || e), "error");
  }
}

async function p2pPull() {
  const addr = document.getElementById("p2pAddrInput")?.value.trim();
  if (!addr) { toast("请输入对端地址", "error"); return; }
  if (!syncState?.private_key_pem) { toast("请先配置私钥（用于解密）", "error"); return; }

  syncLog("正在从 " + addr + " 拉取...", "info");
  try {
    const resp = await fetch("http://" + addr + "/sync");
    if (!resp.ok) throw new Error("HTTP " + resp.status);
    const payload = await resp.json();

    // 解密——用 Rust 端的 decrypt 功能
    // 我们调用 sync_fetch_remote 的替代：直接用 Rust 解密
    // 这里我们把 payload 传给 Rust 解密
    const remote = await invoke("sync_decrypt_payload", { payloadJson: JSON.stringify(payload) });

    pendingRemoteConfig = remote;
    const providers = Object.keys(remote);
    const totalKeys = providers.reduce((sum, p) => sum + (remote[p]?.keys?.length || 0), 0);
    const body = document.getElementById("syncConfirmBody");
    if (body) {
      body.innerHTML = `
        <p>从 ${addr} 获取到：</p>
        <ul style="margin:8px 0;padding-left:20px;">
          <li>${providers.length} 个厂商</li>
          <li>${totalKeys} 个 API Key</li>
        </ul>
        <p style="margin-top:8px;font-size:13px;color:var(--txt-b);">选择同步方式：</p>
      `;
    }
    document.getElementById("syncConfirmModal").style.display = "";
    syncLog("拉取成功", "success");
  } catch (e) {
    const msg = e?.message || e;
    toast("拉取失败: " + msg, "error");
    syncLog("拉取失败: " + msg, "error");
  }
}

async function p2pPush() {
  const addr = document.getElementById("p2pAddrInput")?.value.trim();
  if (!addr) { toast("请输入对端地址", "error"); return; }
  if (!syncState?.public_key_pem) { toast("请先配置公钥（用于加密）", "error"); return; }

  try {
    // 先加密本地配置
    const encryptedJson = await invoke("sync_encrypt_config");

    syncLog("正在推送到 " + addr + "...", "info");
    const resp = await fetch("http://" + addr + "/sync", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: encryptedJson,
    });

    if (!resp.ok) throw new Error("HTTP " + resp.status);
    toast("推送成功");
    syncLog("推送成功", "success");
  } catch (e) {
    const msg = e?.message || e;
    toast("推送失败: " + msg, "error");
    syncLog("推送失败: " + msg, "error");
  }
}

// 处理对端推过来的数据（作为分享方收到数据时）
async function p2pHandleReceived(encryptedJson) {
  if (!syncState?.private_key_pem) {
    syncLog("收到对端推送，但未配置私钥无法解密", "error");
    return;
  }
  try {
    const remote = await invoke("sync_decrypt_payload", { payloadJson: encryptedJson });

    // 自动合并
    await invoke("sync_merge_config", { remote });
    await loadConfig();
    renderAllCards();

    const providers = Object.keys(remote);
    const totalKeys = providers.reduce((sum, p) => sum + (remote[p]?.keys?.length || 0), 0);
    syncLog(`收到对端推送：${providers.length} 个厂商，${totalKeys} 个 Key（已自动合并）`, "success");
    toast("收到对端推送，已合并");
  } catch (e) {
    syncLog("处理推送失败: " + (e?.message || e), "error");
  }
}
```

```rust
/// 解密传入的 payload JSON 字符串
#[tauri::command]
fn sync_decrypt_payload(app: tauri::AppHandle, payload_json: String) -> Result<config::Config, String> {
    let ss = state::load_sync_state(&app);
    let priv_key = ss.private_key_pem.ok_or("未配置私钥")?;
    let payload: sync::EncryptedPayload =
        serde_json::from_str(&payload_json).map_err(|e| format!("解析 payload 失败: {}", e))?;
    sync::decrypt_config(&priv_key, &payload)
}
```

## 用户使用指南

### 场景 1：桌面分享，手机拉取

1. 桌面端打开同步弹窗
2. 确保已生成密钥对（公钥+私钥都有）
3. 点「📤 我来分享」→ 显示二维码和地址
4. 手机端打开同步设置
5. 导入相同的私钥（或用同一套密钥）
6. 输入桌面端的 IP:port
7. 点「⬇ 从对端拉取」→ 选择合并/覆盖

### 场景 2：手机分享，桌面拉取

同上，反过来操作。

### 注意事项

- 两台设备**必须连同一个 WiFi**（同一局域网）
- 两台设备需要**同一套密钥**（私钥用来解密，公钥用来加密）
- 传输的内容已加密，即使局域网被嗅探也安全
- 分享服务**5 分钟无操作或关闭弹窗后自动停止**
